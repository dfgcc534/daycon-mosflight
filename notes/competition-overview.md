# 모기 비행 궤적 예측 AI 경진대회 (DACON 236716)

## 0. 한 줄 요약

LiDAR 로 관측한 **11 개 시점(40 ms 간격)의 모기 3D 좌표** 를 입력받아, **마지막 관측 시점 +80 ms 시점의 (x, y, z) 좌표** 를 예측한다. 평가는 단순 거리 오차가 아닌 **레이저 유효 반경 내 명중률(Hit Rate)** 기반.

## 3. 과제 (Task)

### 3.1 배경

LiDAR 센서 기반의 공중 표적(모기) 실시간 탐지·정밀 조사 시스템에서 **처리 지연**이 발생한다. 이로 인해 조사 시점의 좌표와 탐지 시점의 좌표 사이에 오차가 생기므로, **미래 위치를 예측해 좌표를 보정**해야 한다.

### 3.2 입출력 정의

- **입력**: 40 ms 간격으로 관측된 **11 개 시점**의 모기 3D 좌표
  - 시간축: `-400 ms, -360 ms, ..., -40 ms, 0 ms`
- **출력**: 마지막 관측(0 ms) 기준 **+80 ms** 시점의 (x, y, z) 좌표 1 개

### 3.3 좌표계

- **기준**: 방 기준 절대 좌표 ❌ → **LiDAR 센서 기준 sensor-local 3D 좌표계** ✅
- **단위**: meter (m)
- **축 정의**:
  - `x`: forward (전진)
  - `y`: left (좌측)
  - `z`: up (상향)

### 3.4 환경 다양성

학습/평가 데이터는 **서로 다른 장소·환경**에서 수집되며, 다음 환경들이 섞여 있다:

- 실내, 복도, 창고, 반실외, 야외

→ **도메인 일반화(domain generalization)** 가 핵심 챌린지.

## 5. 평가 지표 (Metric)

### 5.1 두 단계 평가

1. **기본**: 예측 좌표 ↔ 실제 좌표의 **3D Euclidean 거리 오차**
2. **최종 (Hit Rate)**: 예측 좌표가 **레이저 유효 반경 내** 에 들어오는지의 비율

### 5.2 핵심 시사점

> 평균 오차 최소화 ≠ 최적 솔루션
>
> **명중 가능 범위 안에 정확히 들어가는 것** 이 중요. 즉 outlier 1 개의 거리를 줄이는 것보다, 다수 샘플을 hit-radius 안에 가두는 게 유리. → **분포 끝단 (tail) 관리** 가 점수에 큰 영향.

## 7. 솔루션 설계 시사점

본 대회 특성에서 도출한 모델링 방향 (자율 결정용 메모):

1. **시퀀스 길이 11, 타깃 1개** → 매우 짧은 시계열. 무거운 LSTM/Transformer 보다 **delta-based** / **physics-aware** (등가속도 운동 가정) baseline 의 가성비가 높을 가능성.
2. **Hit-Rate metric** → MSE 학습 후 평가하면 mismatch 위험. **Huber / quantile / hit-aware loss** 검토.
3. **도메인 일반화** → scene 정보 미제공이지만, 입력 시퀀스의 **통계 특성**(분산, 평균 속도, 진동 패턴 등)으로 implicit clustering / mixture-of-experts 가능.
4. **노이즈 강함** → 입력에 Kalman/Savitzky-Golay 등 **필터링 전처리** 검토.
5. **외부 데이터 ✅** → 모기 / 곤충 / 작은 표적의 비행 궤적 공개 데이터 탐색 여지.
6. **상대 좌표** (sensor-local) → 데이터 증강 시 평행이동/회전 invariance 활용 (단, 센서 방향 의미가 있어 회전은 신중).
7. **제출 5회/일** → CV 전략을 빡세게 짜야 leaderboard probing 비용 낮춤.
