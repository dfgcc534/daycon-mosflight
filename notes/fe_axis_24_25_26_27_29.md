## Axis 정의

| axis | 의미 |
|---|---|
| 1. sequential only | 시계열 raw 시퀀스 `(B, T, C)` — sample-variant, anchor-invariant, GRU input 형태 |
| 2. sample × anchor | `(B, K, C)` interaction — sample 별 AND anchor 별 모두 다른 값 |
| 3. anchor only | `(K, C)` anchor identity — anchor 별 다름, sample 무관 (batch broadcast) |
| 4. statistic | 시계열 → 압축 stat (EWMA, std, slope 등) — sample-variant, anchor-invariant |
| 5. sample only non-stat | single-step scalar / one-hot / sequential flatten — sample-variant, anchor-invariant, vector 형태 |

## 핵심 진화

- **24** = baseline (seq raw + cand 150). GRU input + attention query.
- **25** = + plan-021 sample-level 134D flatten + macro/EWMA 36 + **seq 8-stat 760D**. LGBM 이라 시계열 raw 못 받음 → 압축.
- **26** = 25 − block ③ 22 (axis 2+3 제거) → 1058D. mode collapse fix lever.
- **27** = 22 + 23 + 26_A2 의 3-way ensemble. FE 추가 없음.
- **29** = 24 + extension 15 (axis 2 sample×anchor 만 강화).

## plan-030 input 후보 (논의용)

- axis 4 (statistic) 의 25 추가본 = **block ④ 760D seq 8-stat + plan-021 macro 9 + EWMA 27** → 796D 가 24/29 paradigm 에 미사용. GRU 가 시계열 직접 처리하므로 redundant 가능성 있으나, attention path 의 query/key 에 *sample-level statistic summary* 로 박으면 sample-variant 강화 가능.
- axis 5 (sample only non-stat) 의 25 추가본 = **L1/L2/L4 134D flatten** → 24/29 에 미사용. attention query 의 sample-level base 확장.
- axis 2 추가 후보: 7 step × anchor projection (plan-029 가 5 step 만 했음).
